# AtlasDoc Release v1 Foundation

## Versioning Policy
- Semantic Versioning (MAJOR.MINOR.PATCH)
- MAJOR: Breaking changes
- MINOR: New features without breaking changes
- PATCH: Bug fixes and minor improvements

## Changelog Policy
- Keep a `CHANGELOG.md` file in the root directory.
- Use markdown format with sections for each version.
- Include dates, descriptions of changes, and contributors.

## Naming Conventions
- Python files: snake_case
- Directories: lowercase_with_underscores
- Classes and Functions: CamelCase
